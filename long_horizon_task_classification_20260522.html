Long Horizon Audit

长程任务分类报告

这份报告按 horizon level 判断数据是否适合训练“当前/历史观测 + 指令 -> latent reasoning -> 未来可执行 IR”。重点区分真正长程 episode / planning、视频或轨迹时序、多视角空间上下文，以及只能作为基础感知能力的单步原子标注。

L3 强长程7 类

episode 控制、Task planning、LeRobot action/state、BDDL demo 与 sequence_plan。

L2 时序/轨迹6 类

视频过程、轨迹点序列、路径选择和空间视频理解。

L1 上下文2 组

多视角、跨图、RGB-depth 和大规模空间图像 QA，但缺少动作序列。

L0 原子能力1 组

单图 bbox、point、segmentation、grounding、QA。

Definition

什么叫长程任务

history future episode trajectory

这里的“长程”不是指文件很大，也不是指 prompt 很长，而是指一个样本的正确输出依赖一段时间、步骤或状态的累积。典型信号包括多帧视频、episode、过去/未来步骤、剩余步骤、next action、action/state 序列、ordered trajectory、sequence_plan 或中间 latent helper image。

算长程的信号

必须根据动作先后、任务阶段、历史/未来、连续视频观察、多个中间推理步骤或完整 episode 才能给出正确答案。

不算长程的信号

只看一张图就能回答一个框、点、分割、单步选择或普通 QA，通常只是长程系统里的基础感知子技能。

核心判断：如果监督目标携带“时间顺序”或“阶段依赖”，按 L2/L3 看；如果只是单个空间标签，即使数据规模很大，也不要误判成长程。

Levels

分级标准

L3显式长程规划 / episode 控制

样本直接描述多步任务、过去/未来步骤、剩余步骤、完整 episode 或 action/state 序列。

RoboInter-Data episodes；MolmoAct2；calvin_lerobot；RoboCerebra demos；RoboInter-VQA Task_planning；ILVR sequence_plan。

L2过程 / 轨迹 / 视频时序

需要按时间顺序理解视频、轨迹点或动作过程，但输出多为局部轨迹、选择或问答。

RoboInter-VQA trajectory；RoboVQA；SIMS/VSI 视频；FSD_visual_trace。

L1多视角 / 跨图上下文

需要比较多张图或 RGB-depth，但没有明确多步行动计划。

SenseNova-SI-8M；RefSpatial 3D multi-view；VST-P multi-image。

L0单步 / 原子标注

单图 bbox、point、segmentation、单问答或单目标定位；可作为长程系统的子技能。

contact box/current box/final box；RoboPoint；SAT；CLEVR；RefCOCO/handal grounding。

Effective Scale

严格长程任务其实不算大

这里按“真正需要跨时间/跨步骤推理”的口径看有效规模，而不是按磁盘大小或总图片数看。

core long-horizon

严格 L3 核心规模

  • RoboInter-Data: 235,880 episodes；87,014,646 frames；471,760 videos。磁盘很大，但 episode 级样本是 23.6 万级。
  • MolmoAct2: 266,539 LeRobot episodes；30,665,227 frames；覆盖 DROID、RT-1、Bridge、BC-Z、LIBERO、MolmoAct tabletop/household，是新加入的核心机器人 action/state 长程数据。
  • calvin_lerobot: 17,870 episodes；1,071,743 frames；389 tasks；适合桌面 manipulation 长程技能补充。
  • RoboCerebra: 1,000 trainset cases + BDDL/demo.hdf5/mp4 + RLDS 分片；更适合做多步桌面/家庭任务 demo 与 benchmark。
  • RoboInter-VQA Task_planning: 965,313 QA records，其中 train 928,819、val 36,494。它是最像 VLM 长程 planning 的主力。
  • ILVR / COMT: 3,850 problems；sequence_plan steps 31,619。推理链清楚，但规模很小。

L2 只是“长程相关”

  • TrajectoryPrediction origin_format train: 130,170；更像局部轨迹/waypoint，不是完整任务规划。
  • TrajChoice/TrajDirection/TrajLang train files: 33,939；FSD_visual_trace: 33,090。
  • SIMS-VSI + VSI-590K video subset 约 577,196 条视频空间样本；SenseNova-SI-8M 有 573 万多图样本，但重点是空间/视角/关系，不是机器人多步执行计划。

结论：如果目标是“长程机器人规划/执行”，核心数据现在应扩展为 RoboInter-Data episode + MolmoAct2 + calvin_lerobot + RoboCerebra demo/benchmark + RoboInter-VQA Task_planning；ILVR 是非机器人推理链补充。SenseNova-SI-8M 与 Molmo2-ER/RefSpatial 更适合作为空间、多视角和视频理解底座。

Boundary Cases

不建议误判为长程的任务

任务分类原因
RoboInter-VQA contact/current/final box多数 L0/L1虽然 final box 预测的是结果位置，但监督目标仍是单个框；缺少多步序列输出。
RoboPoint / refcoco / handal / roborefitL0核心是单图 point/segmentation/grounding，是长程任务中的感知子模块。
RefSpatial 2D/3D choice、SAT、CLEVR、GRiDL0-L1主要是单步空间/深度/关系 QA，不含显式时间计划。
SenseNova-SI-8M 多视角/空间 QAL170.2% 为多图样本，适合视角、方向、距离、时序排序等空间能力；但没有机器人 action/state 序列。
MolmoAct2 / calvin_lerobot / RoboCerebraL3这些是新加入的 episode/demo 数据，不能按普通空间 QA 处理，应保留 episode、frame_index、state/action 或 BDDL/demo 结构。

Training Notes

训练/评估建议

优先用于长程能力

  • 真正的长程机器人能力：优先用 RoboInter-Data、MolmoAct2、calvin_lerobot、RoboCerebra demo/bench，再接 RoboInter-VQA Task_planning 做高层规划监督。
  • 视频过程理解：用 RoboVQA、SIMS-VSI、VSI-590K video subset，以及 MolmoAct2/RoboCerebra 中已对齐的视频资源。
  • 推理链长程：ILVR sequence_plan 可作为非机器人长推理链数据。

作为子技能使用

  • 局部动作轨迹：用 TrajectoryPrediction、TrajChoice、FSD_visual_trace，但不要当完整 planning。
  • 单图 grounding、bbox、point、segmentation、spatial QA 更适合作为长程系统的基础视觉能力。
  • SenseNova-SI-8M 应补 `task_type` 字段后再混训，例如 camera_motion、ego_exo_matching、temporal_order、distance/depth、relative_relation、counting。
  • 建议新增筛选字段：horizon_level、horizon_reason、temporal_dependency、requires_history、requires_future_prediction、output_is_sequence。

Counting Caveats

计数口径注意

  • RoboInter-VQA Generation 的 llava/origin/smart_resize 是格式变体，不能直接相加。
  • VSI-590K 同时包含 video 和 image，只有 video 字段存在的 374,148 条按视频时序处理。
  • ILVR 是 reasoning-chain 长程，不是机器人执行长程。
  • RoboVQA 大 JSON 没有在本报告中重新全量解析 record 数，分类依据来自字段结构、样本 metadata 和既有语义报告。
  • SenseNova-SI-8M 已整理为 JSONL + images 结构；现有分类来自 prompt 模板反推，不是官方显式字段。
  • calvin_lerobot 与 MolmoAct2 的图像/视频形态不同：calvin 图像 bytes 嵌在 parquet，MolmoAct2 多数子集有外部 videos 目录。
